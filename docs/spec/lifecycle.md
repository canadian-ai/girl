# GRP Lifecycle

**Grammar Refactoring Protocol — Artifact Lifecycle Model**

Version 0.1 — Core specification.

## Overview

The GRP lifecycle defines a sequence of artifacts produced during a
refactoring workflow. Each artifact carries identifiers that link it to the
plan that produced it, creating a traceable chain from analysis through
verification.

```
analyze → plan → context pack → refactor → verify
```

## Artifact Types

### Plan

A GRP plan document representing a complete refactoring plan. This is the
primary artifact and the source of truth for the refactoring workflow.

**ID:** `plan.id` — a content-derived deterministic identifier (`grp_xxxxxxxx`).

**Canonical path:** `.grp/plan.json`

**Type:** `dev.refactor.plan`

**Generated by:** `girl plan --output grp-json`

```json
{
  "specversion": "0.1",
  "id": "grp_a1b2c3d4",
  "type": "dev.refactor.plan",
  ...
  "diagnostics": [...],
  "steps": [...],
  "verification": [...]
}
```

### Context Pack

A token-budgeted subset of a plan's source context, designed for consumption
by LLMs or AI agents. References the originating plan via `planId`.

**Relationship:** `contextPack.planId` = `plan.id`

**Canonical path:** `.grp/context-pack.json`

**Type:** `dev.refactor.context`

**Generated by:** `girl pack --output grp-context-json`

```json
{
  "specversion": "0.1",
  "type": "dev.refactor.context",
  "planId": "grp_a1b2c3d4",
  ...
}
```

### Verification Result

A snapshot of the verification commands available for a project at a given
point in time. References the plan via `planId` when provided.

**Relationship:** `verificationResult.planId` = `plan.id` (optional)

**Canonical path:** `.grp/verification.json`

**Type:** `dev.refactor.verification`

**Generated by:** `girl verify --output-file`

```json
{
  "planId": "grp_a1b2c3d4",
  "commands": [...]
}
```

## Artifact Relationships

| Artifact | Field | References |
|----------|-------|-----------|
| Plan | `id` | Self — unique plan identifier |
| Context Pack | `planId` | The plan this context was derived from |
| Verification Result | `planId` | (Optional) The plan this verification belongs to |

## RunID

A `runId` is an opaque string that groups a set of artifacts produced in a
single session. It is not stored in any artifact but MAY be emitted in CLI
output or logs to correlate concurrent workflow stages.

Format: `run_<timestamp>_<random>` (tool-defined).

## File Layout

All GRP artifacts for a project SHOULD be stored under `.grp/` in the
project root or target directory:

```
.grp/
├── plan.json              # Plan artifact
├── context-pack.json      # Context pack artifact
└── verification.json      # Verification result artifact
```

Consumers MUST tolerate missing artifact files. Each artifact is
independently useful — a plan is valid without a context pack.

## Validation

Each artifact type may have its own validation rules:

- **Plan:** Validated by `grp.ValidatePlan()` — requires all required fields,
  correct prefixes, valid enums, repo-relative paths.
- **Context Pack:** Validated by the producer (no formal validation in this
  spec version).
- **Verification Result:** Informational — no formal validation in this spec
  version.

## Edge Cases

| Case | Behavior |
|------|----------|
| Plan not yet generated | `.grp/plan.json` does not exist |
| Context pack without plan | `planId` is empty string |
| Verification before planning | Verification result has no `planId` |
| Multiple runs against same plan | Plan `id` is stable; context packs may differ by `budget` |
| Plan regenerated with same inputs | Plan `id` and content are identical (deterministic) |
