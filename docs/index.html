Local analysis for agent refactoring

Agents should refactor from structure, not vibes.

GIRL is the Grammar-Informed Refactoring Language: a local-first CLI that reads code, detects refactoring pressure, and emits compact GRP plans that AI coding agents can execute safely.

The problem

Prompt-only refactoring is under-specified.

“Clean this up” is not a plan. It hides the important details: which functions are risky, which components own state, what verification is required, and how much context an agent should actually read.

Traditional AST tools know syntax but not the agent workflow. GIRL sits between them: local enough to be private and deterministic, structured enough to guide refactors, and compact enough to keep AI context focused.

GRP

A refactor plan should be machine-readable before it is persuasive.

GRP, the Grammar Refactoring Protocol, is the contract between local analysis and agent implementation. A useful plan names the goal, files, diagnostics, ordered steps, risk, and verification commands.

Design constraints

The repo is built around four constraints.

Local first

No source code upload is required for analysis. The CLI preprocesses code on the developer machine.

Token efficient

The agent gets a plan and selected snippets, not a blind dump of every file in the repository.

Verification bound

Every meaningful refactor should carry the checks needed to prove it did not break behavior.

Recipe driven

Repeated code smells map to reusable refactoring recipes instead of one-off prompt improvisation.

What comes next

Make GIRL precise enough to dogfood hard refactors.

  1. Structured diagnosticsStop parsing human messages; carry symbols, ranges, packages, and metadata explicitly.
  2. Stable step IDsMake generated GRP plans deterministic and safe for multi-step agent execution.
  3. Recipe registryMove diagnostic-to-recipe mapping out of planner switches and into a reusable registry.
  4. Better context packsSelect Go and TypeScript snippets by diagnostic range, symbol importance, and token budget.
  5. Lower-noise analysisImprove heuristics so findings are actionable enough to drive autonomous refactoring.

GIRL is a research-grade engineering tool for safer AI-assisted refactoring.
